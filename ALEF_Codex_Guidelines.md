# ALEF CODEX GUIDELINES

## Principles

1. Never break backwards compatibility.
2. Preserve AERP architecture.
3. One responsibility per module.
4. Use semantic names.
5. Functions must be reusable.
6. Prefer composition over duplication.
7. Every feature must be testable.
8. Never remove public APIs without approval.
9. Prefer declarative design.
10. Code must be Commercial MVP ready.
